Техническое описание Search Engine
Общие принципы работы
Search Engine состоит из двух основных компонентов:

Spider (Паук) - многопоточный веб-краулер, который:

Сканирует веб-страницы начиная с заданного URL

Извлекает текст, заголовки и ссылки из HTML-контента

Индексирует содержимое страниц в базе данных PostgreSQL

Следует за ссылками в пределах заданной глубины обхода

Search Server (Поисковый сервер) - HTTP-сервер, который:

Предоставляет веб-интерфейс для поиска

Обрабатывает поисковые запросы

Возвращает релевантные результаты из базы данных

Поддерживает поиск по 1-4 словам


Архитектура системы
База данных: PostgreSQL с таблицами documents, words и word_frequencies

Обработка текста: Нормализация слов, удаление стоп-слов, подсчет частотности

Сетевые компоненты: HTTP/HTTPS клиент и сервер на Boost.Beast

Многопоточность: Работа с очередью URL в нескольких потоках


Процесс индексации
Загрузка HTML-страницы

Извлечение текстового содержимого и заголовка

Токенизация текста на слова

Нормализация слов (приведение к нижнему регистру, обработка Unicode)

Сохранение в базу данных с подсчетом частотности


Процесс поиска
Парсинг поискового запроса

Нормализация слов запроса

Поиск документов, содержащих все слова запроса

Ранжирование по релевантности (на основе частоты слов)

Формирование HTML-страницы с результатами
